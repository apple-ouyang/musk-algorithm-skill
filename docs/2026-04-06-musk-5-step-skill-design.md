# Musk 5-Step Skill Design

## Objective

Create a reusable agent skill that applies Elon Musk's five-step engineering method as an execution framework for product, engineering, operations, and workflow analysis.

## Scope

The skill should:

- trigger when the user asks to apply Musk's five-step method, The Algorithm, or asks for a ruthless execution/process simplification framework
- help with planning, critique, workflow cleanup, and decision support
- preserve the strict order of the five steps
- use source-backed language from the corpus rather than generic productivity clichés

The skill should not:

- pretend every problem needs automation
- present the five steps as a life-coaching slogan
- overstate weak evidence as primary quotes

## Outputs

When triggered, the skill should produce:

1. a short framing of the target system or process
2. a five-step analysis in order
3. explicit deletions, simplifications, accelerations, and automation gates
4. a final action plan with what to do now, later, and not at all

## References

- Everyday Astronaut interview for the ordered five-step list
- Britannica Elon Musk and xAI pages for timeline context
- Tesla Master Plan and long-form interview sources kept as contextual references even when automated capture failed

## Evaluation

Use prompt-based evals that test:

- correct triggering
- strict step ordering
- refusal to optimize or automate too early
- ability to translate the method into concrete recommendations
